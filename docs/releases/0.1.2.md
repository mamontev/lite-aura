# AuraLite 0.1.2

## Highlights

- Fixed `track my cooldown` auras showing false cooldowns outside combat when other spells were cast.
- Fixed the `Advanced` tab crash and UI recursion/timeout issues affecting cooldown-tracked auras like Shockwave.
- Updated cooldown handling for WoW `12.0.1.66709`, including modern cooldown fields and duration-object rendering support.
- Cleaned up and stabilized the studio UI with improved headers, preview spacing, buttons, inputs, close button styling, and drag/drop tooltip behavior.

## Fixes

- Filtered mirrored GCD/proxy cooldowns so short non-self cooldown noise no longer spawns bogus cooldown rows.
- Hardened cooldown row generation to avoid placeholder rows when no real cooldown exists.
- Reworked UI event dispatch and draft validation to prevent stack overflows and re-entrant editor updates.
- Fixed `Advanced`/editor rendering loops that could trigger `script ran too long`.
- Improved Group Studio and Global Studio layout stability.

## Compatibility

- Updated addon interface support to `12.0.1`.
- Centralized cooldown frame application with support for duration objects and modern cooldown metadata such as `isActive`, `isEnabled`, charges, and LoC replacement data.

## UI Polish

- Removed unnecessary subtitles from the main studio panels.
- Restyled input boxes to better match dropdowns.
- Removed the default red-backed Blizzard close button look in favor of the addon skin.
- Restored the drag-to-spell tooltip on the drop target.

## Verification

- Local automated test suite: `36 passed, 0 failed`.
